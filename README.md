## 项目一：STM32 综合外设与任务实验工程

目录：`2024.8.30_1/`

这是一个面向 STM32 外设综合训练和功能验证的工程，包含多类传感器、执行器、人机交互和通信模块。项目价值不在于单个模块能跑，而在于把多个外设驱动按照相对清晰的目录结构拆分，形成可复用的驱动层、任务层和 UI/交互层代码。

### 项目内容

- **传感器与采集模块**：DHT11 温湿度、BH1750 光照、MAX30102 心率血氧、HX711 称重、超声波测距等
- **身份识别与交互模块**：AS608 指纹识别、RC522 RFID、矩阵键盘、触摸/手势识别、点阵显示、蜂鸣器、LED
- **通信与联网模块**：ESP8266 AT 指令通信、AT 客户端封装、环形缓冲区处理
- **执行控制模块**：舵机、步进电机、PWM、定时器、DAC 等
- **上层任务封装**：`task/`、`Quest/`、`UI/` 中对多个功能进行任务化和界面化组织

### 关键实现

- 将外设驱动按模块拆分为独立 `.c/.h` 文件，便于后续移植和复用
- 对 ESP8266 通信引入 AT 指令封装和 ring buffer，避免串口收发逻辑散落在业务代码中
- 对 MAX30102、RC522、AS608 等典型模块进行独立驱动适配，覆盖 I2C/SPI/UART 等常见总线
- 将底层驱动和上层任务逻辑分离，体现从“能跑”到“能维护”的工程意识

### 能体现的能力

这个工程适合展示我对 STM32 常用外设的广度掌握，以及把传感器、通信、显示和执行机构组合成完整功能链路的能力。对 HR 或面试官来说，它说明我不是只做过单一外设实验，而是有较完整的 MCU 外设联调基础。

## 项目二：STM32 智能车循迹基础控制工程

目录：`循迹基本代码/`

这是一个基于 STM32F1 的智能车循迹控制工程，主要实现红外循迹、按键启动、电机 PWM 控制、左右转向修正、停车/刹车等基础运动控制逻辑。项目虽小，但覆盖了嵌入式控制类项目中非常典型的“传感器输入 - 状态判断 - 执行器输出”闭环。

### 项目内容

- **循迹检测**：通过红外循迹模块采集左右侧路线状态
- **运动控制**：封装小车前进、转向修正、停止等动作接口
- **PWM 输出**：通过 PWM 控制电机速度和运动状态
- **按键启动**：通过按键控制程序进入运行流程，避免上电即动作
- **OLED/LED 辅助显示**：用于状态提示和调试观察

### 核心代码位置

```text
循迹基本代码/User/main.c
循迹基本代码/Hardware/Irtracking.c
循迹基本代码/Hardware/robot.c
循迹基本代码/Hardware/PWM.c
循迹基本代码/Hardware/Key.c
```

### 控制流程

1. 初始化系统时钟、延时、按键、电机、PWM 和循迹模块
2. 等待按键触发，进入小车运行状态
3. 周期读取左右红外循迹状态
4. 根据传感器组合判断直行、左修正、右修正或停止
5. 将判断结果转换为电机控制动作，完成基础闭环控制

### 能体现的能力

这个项目体现了我对嵌入式控制流程的理解：不是只写单个驱动，而是能把输入采集、状态判断和执行控制串成完整逻辑。它也能说明我具备智能车、机器人底盘、基础运动控制类项目的入门实践能力。

## 项目三：TG10 路侧主机核心源码快照

目录：`tg10_source/`

这是一个更接近真实工程场景的嵌入式主机项目源码快照，目标平台为 **STM32H7 + RT-Thread**。项目面向路侧主机设备，涉及启动链路、外设初始化、多链路通信、Lua 脚本接口、参数存储、网络通信预留、OTA 升级和现场故障诊断等内容。

该目录是从完整工程中整理出的干净源码版，保留核心业务代码、板级适配和工程配置，去除了 `build/`、固件产物、Keil 中间文件、完整 SDK 和本地备份目录，便于 GitHub 查看重点代码。

### 项目内容

- **主控平台**：STM32H7，RT-Thread 实时操作系统
- **启动与存储链路**：OSPI Flash、SD、DFS/FatFS、FlashDB、FAL、参数存储
- **通信框架**：UART、RS485、PowerBus 多链路收发，支持接收判帧、协议解析、发送排队、超时重试和异常保护
- **联网能力**：W5500 以太网适配，预留 EC20 4G PPP + MQTT 云端通信链路
- **Lua 脚本能力**：对接 LuatOS/Lua，封装 AIO、DIO、GNSS、LoRa 等接口，支持业务脚本扩展
- **LoRa Mesh 灯控**：封装 Mesh 灯控 API，打通 UDP/串口命令到 Mesh 下发链路，实现灯具模式、频率、亮度等远程控制
- **升级与维护**：Bootloader + FAL OTA、分块写入、CRC 校验、异常恢复、看门狗、复位原因分析、HardFault/Assert 记录

## 为什么这样整理仓库

嵌入式完整工程通常包含大量 SDK、编译产物、工具链文件和本地缓存。如果全部上传，会让仓库非常臃肿，也会淹没真正有价值的业务代码。因此本仓库采用“源码作品集”方式整理：

- 保留能体现开发能力的源码、配置和工程入口
- 排除 `build/`、`Debug/`、`.axf/.bin/.hex/.map`、Keil 中间文件和本地备份
- 对 TG10 项目只上传核心源码快照，不上传完整 SDK 和编译产物

这种整理方式更适合 HR 和技术面试官快速浏览，也更接近实际团队中维护 Git 仓库的方式。
